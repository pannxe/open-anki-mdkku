Version Control

Version control is in this format : A.Bn

A is a main version number, signifies a major iteration (e.g., complete rewrite or platform migration)

B is a sub version number, signifies a minor change (i.e. the main structure of the deck is still preserved, but some cards are changed to improve quality)

n is a letter a-z, signifies minute detail changes (e.g. typo-fixes)

Thus the version number 1.0a is the original one, always the worst, made in rush, a lot of grammatical errors and typo. Later iteration should fix such problems, quality will improve over time.

Change log of each file can be access by click the version number in the tables
